# [TIL] Runway: Netflix의 Model Lifecycle Management 플랫폼 요약

넷플릭스 개인화 인프라 팀의 발표 영상 **"Runway - Model Lifecycle Management at Netflix"**를 보고 정리한 내용. 수천 개의 머신러닝(ML) 모델을 프로덕션에서 안정적으로 운영하기 위한 MLOps 플랫폼 구축 배경과 핵심 인사이트.

---

## 1. 배경 및 기존의 문제점 (The Challenge)
넷플릭스 스케일에서 추천/개인화 모델 수가 기하급수적으로 급증하면서 발생한 인프라적 병목들:

* **인프라 및 툴 파편화 (Fragmented Tooling)**
  * 모델 확인은 모델 레지스트리, 파이프라인 계보(Lineage)는 오케스트레이터, 테스트 현황은 A/B 테스트 플랫폼을 각각 따로 열어서 확인해야 했음.
  * 데이터 과학자의 워크플로우가 파편화되어 생산성이 저하되는 원인이 됨.
* **추적성 상실 (Loss of Traceability)**
  * "이 모델이 어떤 데이터셋과 피처로 학습되었는지", "상위 데이터 파이프라인 변경이 하위 모델에 어떤 영향을 주는지" 전사적으로 파악하기 어려워짐.
* **모델 노후화 (Model Staleness)**
  * 시간이 지나면서 유저 트렌드가 변하고 모델 성능이 떨어지는 현상(Data/Concept Drift)을 실시간으로 감지하고 대응할 체계가 부족했음.

---

## 2. Runway의 핵심 기능과 역할
이 문제를 해결하기 위해 모델 생애주기(Lifecycle) 전반을 관장하는 중앙 메타데이터 서비스 겸 가시성 플랫폼 **'Runway'**를 구축함.

* **통합 메타데이터 및 아티팩트 관리**
  * 학습 데이터셋, 하이퍼파라미터, 환경 설정, 아티팩트 등 모델 관련 모든 데이터를 중앙 레지스트리에서 자동 수집 및 관리.
* **데이터 & 모델 계보(Lineage) 시각화**
  * `[원천 데이터 → 파이프라인 → 피처(Feature) → 모델 → 실험/서빙]` 전 과정을 그래프 형태로 UI에 시각화.
  * 컴포넌트 재사용성을 높이고, 장애 발생 시 파급 효과(Blast Radius)를 쉽게 파악할 수 있음.
* **실시간 비즈니스 피드백 루프 모니터링**
  * 단순 인프라 지표(CPU, Latency)뿐만 아니라, 실제 유저 행동 데이터(Ground Truth, 예: 실제 추천 영상 시청 여부)를 피드백 루프로 삼아 모델 예측값과 실시간 비교.
* **노후화 경고 (Staleness Alerting)**
  * 모델 성능이 임계치 이하로 떨어지거나, 오랫동안 업데이트되지 않으면 담당 팀에 자동 알림을 보내 재학습(Retraining)이나 디버깅을 유도.

---

## 3. 핵심 아키텍처적 인사이트 및 느낀 점

> 💡 *“MLOps의 핵심은 인프라의 복잡성을 숨겨 데이터 과학을 민주화하는 것이다.”*

* **Self-Service 기반의 데이터 과학 민주화**
  * 플랫폼 팀의 개입 없이 데이터 과학자가 스스로 모델 상태를 탐색하고, 의존성을 파악하며, 기존 에셋을 재사용할 수 있게 만드는 것이 대규모 조직에서 병목을 줄이는 핵심인 듯함.
* **비즈니스 메트릭 중심의 QA**
  * 기술적 지표를 넘어 유저 경험과 직결되는 실제 비즈니스 임팩트 메트릭을 모델 생애주기에 녹여내야 실질적인 프로덕션 신뢰도를 보장할 수 있음.
* **가시성(Visibility) = 핵심 인프라**
  * 모델을 빠르게 배포하는 것(Velocity)만큼이나, 거대한 시스템 안에서 모델들이 어떻게 얽혀 있는지 투명하게 들여다볼 수 있는 '거버넌스'와 '추적성'이 장기적인 시스템 안정성에 훨씬 중요함. MLOps에서 시각화와 메타데이터 관리가 왜 최우선순위인지 이해하게 됨.
