## 📝데이터 분석 및 머신러닝 핵심 용어 정리
1. 데이터(Data)의 정의와 형태
데이터는 단순히 숫자가 아니라 **'관찰을 통해 수집된 정보나 특성'**을 의미합니다. 사실상 거의 모든 것을 숫자로 변환할 수 있기 때문에 데이터의 형태는 매우 다양합니다.

다양한 형태: 수치(측정값), 텍스트(단어, 문장, 문서), 이미지, 소리, 비디오 등.

참고: 이미지나 소리도 겉보기엔 숫자가 아닌 것 같지만, 실제로는 픽셀값이나 주파수 같은 숫자로 이루어져 있습니다.

데이터의 관계성:

공간적 관계 (Spatial): 위치나 거리(가깝고 먼 정도)를 기반으로 데이터가 연결됨.

시간적 관계 (Temporal): 시간의 흐름(직전, 한참 전 등)에 따라 데이터가 연결됨.

### 2. 구조에 따른 분류: 정형 vs 비정형 데이터

| 구분 | 정형 데이터 (Structured) | 비정형 데이터 (Unstructured) |
| :--- | :--- | :--- |
| **정의** | 미리 정해진 필드(형식)를 따름 | 고정된 형식이나 필드가 없음 |
| **예시** | Excel, 관계형 데이터베이스(RDB) | 이미지, 비디오, 오디오, 이메일 |
| **특징** | 행(Row)과 열(Column)로 구성됨 | 길이나 형식이 제각각임 |
| **장단점** | 정리/검색/분석이 매우 쉬움 | 전문 도구가 필요하며 분석이 까다로움 |
| **비중** | 보통 조직 내 데이터의 **약 20%** | 보통 조직 내 데이터의 **약 80%** |




## 📝 모델(Model)의 정의와 학습 원리

### 1. 모델이란 무엇인가?

모델은 간단히 말해 둘 이상의 변수 사이의 관계를 **근사치(Approximation)**로 나타낸 것입니다. 

우리가 입력하는 값($x$)과 결과값($y$) 사이의 관계를 함수 형태로 표현한 것이죠.

**기본 수학 공식:**
$$y = f(x) + \epsilon$$

* **$x$ (Input):** 입력 변수 (특성, Feature).
* **$y$ (Output):** 출력 변수 (타겟, Target).
* **$\epsilon$ (Error Term):** 완벽한 모델은 존재할 수 없기 때문에 추가하는 **오차 항목**입니다. 데이터의 무작위성을 100% 예측할 수 없음을 인정하는 지표예요.



---

### 2. 구체적인 모델 예시

강의에서는 모델을 설명하기 위해 두 가지 아주 좋은 예시를 들었습니다.

| 원문 (English) | 한국어 번역 및 요약 | 핵심 포인트 |
| :--- | :--- | :--- |
| **Fuel efficiency from horsepower...** | 엔진 마력($x$) 데이터를 가지고 연비($y$)를 예측하는 모델을 만들 수 있습니다. | 입력($x$)과 출력($y$)의 관계 설정 |
| **Model can roughly approximate... but not perfectly capture randomness.** | 모델이 연비를 대략적으로 맞출 순 있어도, 데이터의 무작위성(Randomness)까지 완벽히 잡을 순 없어요. | 항상 **오차(Error)**가 존재함을 명시 |
| **Predict the sale price of homes...** | 집의 면적, 방 개수, 학군($x$) 등을 이용해 집값($y$)을 예측하는 모델도 마찬가지입니다. | 과거 데이터를 통한 예측 모델 구축 |

---

### 3. 모델을 만들기 위해 정의해야 할 4가지 요소

머신러닝에서 모델을 설계할 때 우리가 직접 결정해야 하는 4가지 핵심 요소입니다.

1.  **특성 (Features):** 모델의 입력값($x$)으로 사용할 데이터의 속성들 (예: 평수, 학군).
2.  **알고리즘 (Algorithm):** 모델의 전체적인 '틀'이나 '템플릿'입니다. 모델의 대략적인 형태와 구조를 결정해요.
3.  **하이퍼파라미터 (Hyperparameters):** 알고리즘에 달린 **조절 노브(Knob)**라고 생각하면 됩니다. 모델을 더 단순하게 혹은 더 복잡하게 만들어서 문제에 최적화하는 설정값입니다.
4.  **손실 함수 (Loss Function):** 모델의 오차(Error)를 수치화하는 기준입니다. 이 수치가 낮을수록 좋은 모델이겠죠?



---

### 4. 모델 학습(Training)의 핵심

모델을 '학습'시킨다는 것은 결국 **손실 함수(Loss Function)를 최소화**하는 과정입니다.

* **학습 과정:** 과거의 입력값($x$)과 실제 결과값($y$)을 모델에 제공합니다.
* **오차 최소화:** 우리가 선택한 알고리즘과 하이퍼파라미터 구조 안에서, 모델이 내놓는 예측값과 실제 정답 사이의 차이(손실)를 가장 작게 만드는 내부 수치들을 찾아냅니다.
* **결과:** 손실 함수가 최소가 되는 지점에 도달하면, 비로소 예측력이 높은 모델이 완성됩니다.

---

### 핵심 요약 (Key Takeaways)

1.  **모델은 완벽하지 않다:** 항상 오차($\epsilon$)가 존재한다는 것을 전제로 합니다.
2.  **4대 구성 요소:** 재료(특성), 레시피(알고리즘), 설정값(하이퍼파라미터), 평가 기준(손실 함수)이 모두 필요합니다.
3.  **학습 = 최적화:** 과거 데이터를 사용해 손실(Loss)을 최소화하는 최적의 지점을 찾는 것이 학습의 본질입니다.



## 📝 머신러닝의 3가지 주요 유형 (Primary Types of Machine Learning)

머신러닝은 크게 **지도 학습, 비지도 학습, 강화 학습** 세 가지로 나뉩니다. 각 유형은 해결하려는 문제의 성격과 데이터의 형태에 따라 구분됩니다.

---

### 1. 지도 학습 (Supervised Learning)

지도 학습의 목표는 주어진 데이터를 바탕으로 **타겟 변수($y$)를 예측**하는 것입니다. 마치 선생님이 정답을 알려주듯, 과거의 정답(Label)이 포함된 데이터를 학습합니다.



* **핵심 특징:** 과거 관측 데이터와 그에 매칭되는 '정답(Target)'이 모두 필요합니다.
* **주요 작업:**
    * **분류 (Classification):** 사물의 범주나 클래스를 인식 (예: 사과 사진 보고 '사과'라고 맞추기).
    * **회귀 (Regression):** 수치형 변수를 예측 (예: 집값 예측).
* **현실 예시:**
    * X-ray 이미지를 보고 폐렴 여부 진단하기.
    * 부동산 데이터를 기반으로 집값 예측하기.
    * 이메일이 스팸인지 아닌지(Binary Classification) 구분하기.

---

### 2. 비지도 학습 (Unsupervised Learning)

비지도 학습은 정답이 없는 상태에서 **데이터 자체의 내재된 구조나 패턴**을 찾아내는 것이 목표입니다.



* **핵심 특징:** 관측 데이터는 있지만, 그에 따른 '정답(Target)'은 없습니다.
* **주요 작업:**
    * **클러스터링 (Clustering):** 비슷한 특성을 가진 것들끼리 묶기.
    * **이상 탐지 (Anomaly Detection):** 평소와 다른 패턴 찾아내기.
* **현실 예시:**
    * **시장 세분화 (Market Segmentation):** 고객의 구매 패턴을 분석해 논리적인 그룹으로 나누기.
    * **과일 분류:** 사과, 오렌지, 바나나 사진을 주면 이름은 모르더라도 색상이나 모양을 보고 끼리끼리 묶어냄.

---

### 3. 강화 학습 (Reinforcement Learning)

강화 학습은 환경과의 상호작용을 통해 **목표를 달성하기 위한 전략**을 배우는 방식입니다.

* **핵심 특징:** 정답 데이터를 주는 대신, 행동에 따른 보상(Reward)을 통해 최선의 방법을 학습합니다.
* **현실 예시:**
    * 컴퓨터에게 체스나 체커 같은 게임의 필승 전략을 가르칠 때 주로 사용됩니다.

---

### 지도 학습 vs 비지도 학습 비교 (과일 예시)

| 구분 | 지도 학습 (Supervised) | 비지도 학습 (Unsupervised) |
| :--- | :--- | :--- |
| **데이터 구성** | 사진 + "이것은 사과다"라는 레이블 | 레이블 없는 수많은 과일 사진들 |
| **모델의 역할** | 새로운 사진을 보고 "이것은 사과다"라고 이름표를 붙임 | "이것들은 빨갛고 둥그니까 A그룹"이라고 패턴별로 묶음 |
| **목표** | 특정 출력(Target)을 **예측**하는 것 | 데이터를 **조직화/그룹화**하는 것 |

---

### 지도 학습의 상세 분류: 회귀 vs 분류



| 종류 | 목표 | 예시 |
| :--- | :--- | :--- |
| **회귀 (Regression)** | **연속적인 수치** 예측 | 집값, 신제품 수요량, 전력 소모량 예측 |
| **분류 (Classification)** | **범주/카테고리** 식별 | 질병 진단(유/무), 식물 종류 판별, 스팸 메일 차단 |

* **참고:** 분류 문제는 'Yes/No' 같은 이진 분류(Binary)일 수도 있고, 꽃의 종류처럼 여러 카테고리 중 하나를 고르는 다중 분류일 수도 있습니다.

### 💡 핵심 요약 (Key Takeaways)

1.  **지도 학습:** 정답($y$)이 있는 데이터를 통해 **분류**하거나 **수치**를 맞춤.
2.  **비지도 학습:** 정답 없이 데이터의 **패턴**을 찾아 그룹화함 (클러스터링).
3.  **강화 학습:** 시행착오와 보상을 통해 최적의 **전략**을 수립함.
4.  **실무 적용:** 예측하고 싶은 값이 숫자면 **회귀**, 카테고리면 **분류** 알고리즘을 선택함.




## 📝 머신러닝의 가능성과 한계 (What ML Can and Cannot Do)

머신러닝을 실생활에 잘 적용하려면 기술의 작동 원리뿐만 아니라, **어떤 상황에서 잘 작동하고 어떤 상황에서 한계가 있는지**를 명확히 아는 것이 중요합니다.

---

### 1. 머신러닝이 잘 작동하기 위한 전제 조건
머신러닝은 만능이 아니며, 다음 조건이 충족될 때만 제 성능을 발휘합니다.
* **충분한 양의 데이터 (Quantity):** 학습할 샘플이 너무 적으면 패턴을 찾을 수 없습니다.
* **높은 데이터 품질 (Quality):** 데이터가 지저분하거나(Noisy), 이상치(Outlier)가 많거나, 결측치가 많으면 아무리 좋은 알고리즘도 소용없습니다.

---

### 2. 머신러닝이 잘하는 것 (Strengths)



| 주요 기능 | 설명 | 구체적인 예시 |
| :--- | :--- | :--- |
| **단순 작업 자동화** | 사람이 반복적으로 하는 명확한 작업을 대신 수행 | 우편물 주소 판독 및 분류, 실시간 강의 자동 자막 생성(Zoom 등) |
| **입출력 관계 예측** | 입력값과 결과값 사이의 복잡한 상관관계를 학습해 예측 | 마력에 따른 연비 예측, 출석률/과제 점수에 따른 학생 성적 예측 |
| **개인화 서비스** | 개별 사용자의 특성에 맞춰 제품이나 서비스를 맞춤 제공 | **Netflix** 영화 추천, **Amazon** 구매 이력 기반 상품 추천 |

---

### 3. 머신러닝이 못하는 것 (Limitations)

머신러닝 모델을 다룰 때 가장 주의해야 할 한계점들입니다.

####  맥락(Context) 파악 불가
* 사람은 대화 중 **농담이나 비꼬는 말(Sarcasm)**을 맥락으로 이해하지만, 모델은 모든 말을 **문자 그대로(Literal)** 해석합니다.
* 번역기 모델이 단어는 맞게 옮겨도 상황에 어색한 번역을 내놓는 이유가 여기에 있습니다.

####  인과관계(Causation) 판단 불가
* **상관관계(Correlation) vs 인과관계:** 

    **가장 중요한 주의사항:** ML은 데이터 간의 **패턴과 상관관계**만 식별할 뿐, 'A가 B의 원인이다'라는 **인과관계**를 증명하지 못합니다.
* **예시:** 여름철에 '아이스크림 판매량'과 '범죄율'은 둘 다 올라가서 상관관계가 높게 나오지만, 아이스크림이 범죄의 원인은 아닙니다. (진짜 원인은 '여름'이라는 계절성입니다.)



####  '왜' 일어나는지 설명 불가
* 모델은 "이런 입력들이 들어왔을 때 이런 결과가 나올 확률이 높다"라고 패턴을 제시할 뿐, **왜 그런 결합이 그런 결과를 만드는지** 논리적인 이유를 스스로 설명하지 못합니다.

#### ④ 해결책 제시 및 개입 효과 측정 불가
* **문제 해결의 부재:** 범죄 발생 지역을 정확히 예측하는 모델을 만들 수는 있지만, 그 모델이 "범죄를 줄이려면 어떤 법을 만들어야 하는가?"라는 질문에는 답을 주지 못합니다.
* **개입의 영향:** 특정 정책(예: 총기 규제 등)이 결과에 어떤 영향을 미칠지 미리 시뮬레이션하거나 스스로 해결책을 제안할 수 없습니다.

---

### 💡 핵심 요약 (Key Takeaways)

1.  **Garbage In, Garbage Out:** 데이터의 양과 질이 받쳐주지 않으면 머신러닝은 단순한 작업 조차 수행하지 않습니다.
2.  **강점:** ML은 상관관계를 찾는 반복 작업 자동화, 수치 예측, 개인화 추천에 매우 강력합니다.
3.  **약점:** 하지만 **맥락, 비유, 인과관계**를 이해하지 못하며, 문제에 대한 **해결책(Solution)**을 스스로 제안하지 못합니다.
4.  **주의사항:** 데이터상에서 발견된 상관관계를 곧바로 원인과 결과로 오해해서는 안 됩니다. 모델은 '현상'을 예측할 뿐, 문제를 해결하기 위한 '처방(Solution)'을 내리지는 않습니다. 그 결정은 인간의 몫입니다.
