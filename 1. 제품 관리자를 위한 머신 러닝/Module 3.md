# Module 3)모델 해석과 평가


지난 모듈에서는 교차 검증을 포함해 머신러닝 모델을 구축하고 훈련하는 전반적인 과정을 다루었습니다. 이번 새로운 모듈에서는 완성된 모델의 결과물을 **어떻게 해석하고, 어떤 지표로 정확하게 평가할 것인지**에 대해 훨씬 더 깊이 있게 파고듭니다.

---

### 1. 이번 모듈의 핵심 방향

모델의 성능을 단순히 '좋다/나쁘다'로 뭉뚱그려 판단하는 것을 넘어, 비즈니스 목적과 데이터의 성격에 맞는 정확한 수치 지표를 사용하는 방법을 학습합니다.

* **지표의 세분화:** 성과를 측정하는 관점에 따른 지표 분류.
* **지도 학습(Supervised Learning)의 두 갈래:** 문제 유형에 따른 맞춤형 평가 방법론 적용.

### 2. 학습 목표

## 📝 결과 지표 vs 출력 지표 (Outcome vs. Output Metrics)

모델을 평가하고 해석하는 작업은 CRISP-DM 프로세스의 5단계(평가)에 해당하지만, 사실 그 **평가의 기준을 세우는 일은 1단계인 비즈니스 이해(Business Understanding)에서부터 시작**됩니다. 모델이 '성공적'이라는 것을 어떻게 정의할지 명확히 해야 하기 때문입니다.

머신러닝에서는 모델의 성공을 측정하기 위해 크게 두 가지 유형의 지표를 사용합니다.

---

### 1. 결과 지표와 출력 지표의 핵심 차이

고객이 진정으로 원하는 것과, 엔지니어가 모델을 튜닝하기 위해 보는 지표는 완전히 다릅니다. 이 둘을 명확히 구분하는 것이 프로젝트 성공의 핵심입니다.



| 구분 | 결과 지표 (Outcome Metrics) | 출력 지표 (Output Metrics) |
| :--- | :--- | :--- |
| **정의** | 모델이나 제품이 창출하는 **실제 비즈니스 임팩트** | 모델 자체의 **기술적 예측 성능** |
| **주요 대상** | 고객, 비즈니스 이해관계자, 경영진 | 데이터 과학자, 머신러닝 엔지니어 |
| **측정 단위** | 달러(비용 절감, 수익 창출), 시간, 사고 건수 등 | 정확도, 오차율 등 (이후 강의에서 학습) |
| **결정 순서** | 프로젝트 초기에 가장 먼저 정의됨 | 결과 지표가 정해진 후, 이를 달성하기 위해 선택됨 |
| **특징** | 기술적인 모델 성능 수치를 전혀 포함하지 않음 | 고객에게는 거의 공유되지 않음 |

---

### 2. [사례 연구] 지표 적용의 실제

이 두 지표가 실무에서 어떻게 다르게 적용되는지 두 가지 비즈니스 사례를 통해 확인해 보겠습니다.

#### 사례 1: 항공사 난기류 예측 모델 (분류 문제)
* **목표:** 이륙 전 기상 조건을 분석하여 난기류를 예측하고 안전한 비행경로를 최적화.
* **결과 지표 (Outcome):**
  * 연간 안전사고 발생 건수 감소
  * 안전사고로 인한 항공사 배상 청구액(달러) 감소
* **출력 지표 (Output):** 모델이 난기류 유무를 얼마나 잘 맞추는지 평가하는 기술적 **분류(Classification) 지표**.

#### 사례 2: 전력 회사의 전력 수요 예측 모델 (회귀 문제)
* **목표:** 전력 수요를 정확히 예측하여 발전량을 최적화하고, 비싸고 탄소 배출이 많은 피커 발전소(Peaker Plants)의 가동을 최소화.
* **결과 지표 (Outcome):**
  * 생산된 메가와트시(MWh)당 발전 비용 감소
  * 생산된 메가와트시(MWh)당 탄소 배출량 감소
* **출력 지표 (Output):** 모델이 실제 수요량을 수치적으로 얼마나 근접하게 예측하는지 평가하는 기술적 **회귀(Regression) 지표**.



---

### 💡 핵심 요약 (Key Takeaways)

1. **시작과 끝의 연결:** 1단계(비즈니스 이해)에서 세운 '결과 지표'가 5단계(평가)에서 모델의 '출력 지표'를 결정하는 나침반이 됩니다.
2. **고객의 관심사:** 고객은 모델의 정확도(Output)가 아니라, 그 모델이 내 지갑을 얼마나 두껍게 해 주는지, 내 비용을 얼마나 줄여주는지(Outcome)에만 관심이 있습니다.
3. **지표의 종속성:** 항상 비즈니스 결과 지표(Outcome)를 먼저 설정하고, 이를 가장 잘 반영할 수 있는 기술적 출력 지표(Output)를 나중에 선택해야 합니다.

---

## 📝 모델 출력 지표의 활용 시점 (Using Model Output Metrics)

출력 지표(Output Metrics)는 단순히 최종 성적표로만 쓰이는 것이 아닙니다. 모델을 만들고, 검증하고, 실제 서비스에 적용한 후까지 **모델의 전체 생애 주기(Lifecycle) 전반에 걸쳐 세 번의 핵심적인 시점**에 사용됩니다.

---

### 1. 출력 지표를 확인하는 3가지 핵심 단계

모델링 워크플로우를 따라가며 출력 지표가 언제, 어떤 목적으로 쓰이는지 정리해 보겠습니다.

| 진행 단계 | 목적 | 상세 내용 |
| :--- | :--- | :--- |
| **① 모델 선택 및 튜닝 (Pre-selection)** | **비교 및 평가** | 여러 알고리즘과 하이퍼파라미터를 적용해 본 다양한 후보 모델들의 성능을 비교하여, **최적의 모델을 선택(Select)**하기 위해 지표를 확인합니다. |
| **② 배포 전 최종 평가 (Pre-deployment)** | **실전 성능 검증** | 최종 선택된 모델을 고객에게 제공(배포)하기 직전, 이 모델이 실제 환경에서 얼마나 잘 작동할지 **최종 성적을 매기기 위해** 지표를 확인합니다. |
| **③ 배포 후 모니터링 (Post-deployment)** | **지속적인 품질 관리** | 모델이 현실 세계(Real world)에 배포되어 작동하는 동안, 데이터 변화 등에 의해 성능이 떨어지지 않는지 **지속적으로 추적하고 평가**하기 위해 지표를 확인합니다.



---

### 2. 단계별로 사용하는 데이터 세트의 차이

지난 모듈에서 배운 '데이터 분할 전략'과 '교차 검증'이 바로 이 출력 지표를 계산할 때 적용됩니다. 평가의 목적에 따라 **어떤 데이터로 지표를 계산할 것인가**가 명확히 구분되어야 합니다.

* **모델을 비교하고 선택할 때 (① 단계):**
    * 사용 데이터: **검증 세트 (Validation Set)** 또는 **교차 검증 (Cross-Validation)** 전략 사용.
    * 이유: 테스트 세트를 오염시키지 않고(데이터 누수 방지) 여러 모델을 공정하게 모의고사 치르기 위함입니다.
* **최종 모델을 평가할 때 (② 단계):**
    * 사용 데이터: **테스트 세트 (Test Set)**.
    * 이유: 모델 학습과 튜닝 과정에서 단 한 번도 사용되지 않은, 완전히 새로운 데이터(Unseen data)를 통해 편향 없는(Unbiased) 최종 예측 성능을 확인하기 위함입니다.

---
## 📝 회귀 모델 평가 지표 (Regression Error Metrics)

연속적인 숫자를 예측하는 회귀(Regression) 모델의 성능을 평가할 때는 예측값과 실제 정답이 얼마나 차이 나는지, 즉 **오차(Error)**를 측정합니다. 실무에서 가장 널리 쓰이는 4가지 핵심 평가 지표를 수학적 의미와 함께 정리해 드립니다.



### 1. MSE (Mean Squared Error)와 RMSE

가장 유명하고 널리 쓰이는 지표로, 실제값과 예측값의 차이를 제곱하여 평균을 낸 값입니다.

$$MSE = \frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2$$

* **특징:** 오차를 '제곱'하기 때문에, 단 하나의 큰 오차(Outlier)가 발생해도 페널티를 아주 강하게 부여합니다.
* **단점:** 데이터의 단위(Scale)에 영향을 받으므로 다른 문제의 모델을 서로 비교하기 어렵고, 제곱으로 인해 직관적인 수치 해석이 까다롭습니다.
* **RMSE (Root Mean Squared Error):** MSE의 해석이 어려운 단점을 보완하기 위해 루트를 씌운 값으로, 원래 데이터와 단위가 같아집니다.

$$RMSE = \sqrt{MSE}$$

---

### 2. MAE (Mean Absolute Error)

오차의 '절댓값'을 모두 더해 평균을 낸 지표입니다.

$$MAE = \frac{1}{n} \sum_{i=1}^{n} |y_i - \hat{y}_i|$$

* **특징:** 제곱을 하지 않기 때문에 이상치(Outlier)에 덜 민감합니다.
* **장점:** 예측하려는 타겟 데이터와 동일한 단위를 가지므로 결과를 직관적으로 이해하기 쉽습니다. (예: 집값 예측에서 MAE가 1000이라면 평균적으로 1000달러 정도 틀린다는 의미입니다.)

---

### 3. MAPE (Mean Absolute Percentage Error)

오차를 절댓값으로 구한 뒤 실제값으로 나누어 **백분율(%)**로 변환한 지표입니다.

$$MAPE = \frac{1}{n} \sum_{i=1}^{n} \left| \frac{y_i - \hat{y}_i}{y_i} \right| \times 100$$

* **장점:** 결과가 퍼센트로 나오기 때문에 비즈니스 부서나 비기술직 고객에게 설명하기 가장 좋습니다.
* **단점:** 실제값이 0에 가깝거나 아주 작을 경우, 오차가 조금만 발생해도 분모가 작아져 지표 값이 비정상적으로 폭증하는 현상이 발생합니다.

---

### 4. 실무 비교: MSE vs MAE

총 오차의 합이 같더라도 지표에 따라 평가가 완전히 달라집니다. 총 오차 합이 7인 두 가지 케이스를 비교해 보겠습니다.

| 오차 분포 현황 | MAE 계산 | MSE 계산 | 실무 적용 추천 가이드 |
| :--- | :--- | :--- | :--- |
| **Case 1:** 자잘하게 계속 틀림 (1, 2, 1, 2, 1) | 1.4 | **2.2** | 평소의 자잘한 오차를 줄이는 것이 중요할 때는 **MAE** 사용 |
| **Case 2:** 다 맞추고 딱 한 번 크게 틀림 (0, 0, 0, 0, 7) | 1.4 | **9.8** | 단 한 번의 큰 오차가 비즈니스에 치명적일 때는 페널티가 강한 **MSE** 사용 |

---

### 5. R-squared (결정계수)

앞선 지표들이 '오차가 얼마나 큰가'를 쟀다면, R-squared는 모델이 **데이터의 변동성을 얼마나 잘 설명하는가**를 0과 1 사이의 점수로 나타냅니다.



* **SST (총 변동성):** 데이터가 평균으로부터 얼마나 퍼져있는지 나타냅니다.
* **SSR (설명된 변동성):** 모델의 예측값이 평균으로부터 얼마나 퍼져있는지 나타냅니다.
* **SSE (오차):** 모델이 설명하지 못한 변동성(실제값과 예측값의 차이)입니다.

$$R^2 = \frac{SSR}{SST} = 1 - \frac{SSE}{SST}$$

* **해석:** 1에 가까울수록 완벽하게 패턴을 설명하는 모델이며, 0에 가까울수록 설명력이 전혀 없는 모델입니다.

## 📝 분류 모델 평가 지표: 혼동 행렬 (Classification Error Metrics)

회귀 모델이 '오차'를 측정했다면, '스팸이다/아니다', '양성/음성'을 나누는 분류(Classification) 모델은 **'얼마나 정확하게 분류했는가'**를 측정합니다. 이번 레슨에서는 분류 모델의 가장 핵심적인 평가 도구인 **혼동 행렬(Confusion Matrix)**과 주요 지표들을 정리합니다.

---

### 1. 정확도(Accuracy)의 함정: 클래스 불균형

가장 직관적이고 흔히 쓰이는 지표는 **정확도(Accuracy)**입니다. 전체 예측 중에서 맞춘 예측의 비율을 의미합니다. 하지만 정확도에는 치명적인 약점이 있습니다.

* **문제 상황 (클래스 불균형, Class Imbalance):** 환자의 99.4%가 정상이고 0.6%만 심장병을 가진 데이터가 있다고 가정해 보겠습니다. 
* **결과:** 모델이 아무 분석도 하지 않고 무조건 "모든 환자는 정상(0)이다"라고만 예측해도 이 모델의 정확도는 99.4%가 됩니다. 수치상으로는 훌륭해 보이지만, 실제로는 병을 하나도 찾아내지 못하는 쓸모없는 모델입니다.

이러한 착시를 방지하기 위해 **혼동 행렬**을 사용합니다.

---

### 2. 혼동 행렬 (Confusion Matrix)

혼동 행렬은 모델이 무엇을 맞추고 무엇을 헷갈려(Confusion) 하는지 4가지 케이스로 나누어 보여주는 표입니다.



| 실제 값 (True) \ 예측 값 (Predicted) | 예측: 양성 (1) | 예측: 음성 (0) |
| :--- | :--- | :--- |
| **실제: 양성 (1)** | **TP (True Positive):** 병이 있는데, 있다고 맞춤 | **FN (False Negative):** 병이 있는데, 없다고 틀림 |
| **실제: 음성 (0)** | **FP (False Positive):** 병이 없는데, 있다고 틀림 | **TN (True Negative):** 병이 없는데, 없다고 맞춤 |

---

### 3. 혼동 행렬에서 파생되는 3가지 핵심 지표

정확도의 한계를 극복하기 위해, 혼동 행렬의 값을 조합하여 비즈니스 목적에 맞는 세부 지표를 계산합니다.



[Image of Precision vs Recall in Machine Learning]


#### ① 재현율 (Recall) 또는 민감도 (Sensitivity) / TPR
**"실제 양성(1)인 데이터 중에서, 모델이 얼마나 양성으로 잘 찾아냈는가?"**
놓치면 안 되는 중요한 케이스(예: 암 환자 진단)를 평가할 때 가장 중요한 지표입니다.

$$Recall = \frac{TP}{TP + FN}$$

#### ② 정밀도 (Precision)
**"모델이 양성(1)이라고 예측한 것 중에서, 진짜 양성인 비율은 얼마나 되는가?"**
잘못된 예측으로 인한 비용이 클 때(예: 정상 메일을 스팸으로 분류해 삭제해버리는 경우) 중요한 지표입니다.

$$Precision = \frac{TP}{TP + FP}$$

#### ③ 거짓 양성 비율 (FPR: False Positive Rate)
**"실제 음성(0)인 데이터 중에서, 모델이 양성으로 잘못 예측한 비율은 얼마나 되는가?"**

$$FPR = \frac{FP}{FP + TN}$$

---

### 4. 다중 클래스 혼동 행렬 (Multi-class Confusion Matrix)

이진 분류(0 또는 1)뿐만 아니라, 3개 이상의 클래스(예: 고양이, 개, 새)를 분류할 때도 혼동 행렬을 확장하여 사용할 수 있습니다.

* **장점:** 모델이 유독 '개'와 '고양이'를 헷갈려한다는 식의 구체적인 약점을 시각적으로 쉽게 파악할 수 있습니다.
* **평가 방식:** 각 클래스별로 정밀도와 재현율을 구한 뒤, 이를 평균 낸 **매크로 평균(Macro-averaged)** 지표를 사용하여 전체 성능을 평가합니다.

---

클래스 불균형에 속지 않고 모델의 진짜 실력을 파악할 수 있는 혼동 행렬과 정밀도, 재현율의 개념을 정리했습니다! 

이 지표들을 활용해 모델의 성능을 시각적인 그래프로 보여주는 **ROC 커브와 AUC**에 대해 바로 이어서 학습해 볼까요? 아니면 이 내용들에 대해 확인해 볼 수 있는 간단한 퀴즈를 하나 내드릴까요?

## 📝 분류 모델 평가 지표: 혼동 행렬 (Classification Error Metrics)

회귀 모델이 '오차'를 측정했다면, 분류(Classification) 모델은 양성/음성, 스팸 여부 등을 **'얼마나 정확하게 분류했는가'**를 측정합니다. 이번 레슨에서는 분류 모델의 가장 핵심적인 평가 도구인 **혼동 행렬(Confusion Matrix)**과 주요 지표들을 정리합니다.

---

### 1. 정확도(Accuracy)의 계산법과 클래스 불균형의 함정

가장 직관적이고 흔히 쓰이는 지표는 **정확도(Accuracy)**입니다. 전체 예측 중에서 올바르게 맞춘 예측의 비율을 의미합니다.

$$Accuracy = \frac{\text{맞춘 예측 수 (Correct Predictions)}}{\text{전체 예측 수 (Total Predictions)}}$$

하지만 정확도에는 **클래스 불균형(Class Imbalance)** 상황에서 모델의 성능을 속일 수 있는 치명적인 약점이 있습니다.

* **문제 상황:** 수천 명의 환자 데이터 중 99.4%가 정상(0)이고 0.6%만 심장병(1)을 가진 상태입니다.
* **결과:** 모델이 아무 학습도 하지 않고 무조건 "모든 환자는 정상(0)이다"라고만 찍어도, 이 모델의 정확도는 99.4%가 됩니다. 수치상으로는 훌륭해 보이지만 실제로는 병을 하나도 찾아내지 못하는 쓸모없는 모델입니다.

이러한 착시를 방지하고 모델의 진짜 실력을 평가하기 위해 **혼동 행렬**을 사용합니다.

---

### 2. 혼동 행렬 (Confusion Matrix)

혼동 행렬은 한 축에는 실제 값(True)을, 다른 축에는 모델의 예측 값(Predicted)을 두고, 모델이 무엇을 맞추고 무엇을 헷갈려(Confusion) 하는지 4가지 경우의 수로 나누어 보여주는 표입니다.



| 실제 값 (True) \ 예측 값 (Predicted) | 예측: 양성 (1) | 예측: 음성 (0) |
| :--- | :--- | :--- |
| **실제: 양성 (1)** | **TP (True Positive):** 병이 있는데, 있다고 맞춤 | **FN (False Negative):** 병이 있는데, 없다고 틀림 |
| **실제: 음성 (0)** | **FP (False Positive):** 병이 없는데, 있다고 틀림 | **TN (True Negative):** 병이 없는데, 없다고 맞춤 |

---

### 3. 혼동 행렬에서 파생되는 3가지 핵심 에러 지표

정확도의 한계를 극복하기 위해, 혼동 행렬의 4가지 값을 조합하여 각기 다른 관점의 성능 지표를 계산합니다.



[Image of Precision vs Recall in Machine Learning]


#### ① 진양성률 (TPR) / 재현율 (Recall) / 민감도 (Sensitivity)
**"실제 양성(1)인 전체 데이터 중에서, 모델이 양성으로 올바르게 찾아낸 비율"**
(예: 실제 심장병 환자 중 몇 명이나 병을 찾아냈는가?)

$$Recall = \frac{TP}{TP + FN}$$

#### ② 거짓 양성 비율 (FPR: False Positive Rate)
**"실제 음성(0)인 전체 데이터 중에서, 모델이 양성이라고 잘못 예측한 비율"**
(예: 정상인 중에서 몇 명이나 병이 있다고 오진했는가?)

$$FPR = \frac{FP}{FP + TN}$$

#### ③ 정밀도 (Precision)
**"모델이 양성(1)이라고 예측한 것 중에서, 실제로 진짜 양성인 비율"**
(예: 병이 있다고 진단받은 사람 중, 진짜로 병이 있는 사람은 몇 명인가?)

$$Precision = \frac{TP}{TP + FP}$$

---

### 4. 다중 클래스 혼동 행렬 (Multi-class Confusion Matrix)

이진 분류(0 또는 1)뿐만 아니라, 3개 이상의 다중 클래스(예: 고양이, 개, 새)를 분류할 때도 각 축의 클래스 개수를 늘려 혼동 행렬을 똑같이 적용할 수 있습니다.

* **장점:** 모델이 유독 '개'와 '고양이'를 헷갈려한다는 식의 구체적인 약점을 표를 통해 한눈에 파악할 수 있습니다.
* **평가 방식:** 전체 성능을 하나로 요약하기 위해, 각 클래스별로 정밀도와 재현율을 먼저 구한 뒤 이를 평균 낸 **매크로 평균 재현율(Macro-averaged recall)**과 **매크로 평균 정밀도(Macro-averaged precision)**를 계산하여 사용합니다.
---
## 📝 분류 모델 평가 지표 심화: ROC 커브와 PR 커브 (ROC and PR Curves)

분류 모델의 성능을 시각적으로 평가하고 최적의 기준점을 찾기 위해 실무에서 가장 자주 활용되는 두 가지 그래프, **ROC 커브**와 **PR 커브**에 대해 알아보겠습니다.

---

### 1. 임계값(Threshold)의 이해

그래프를 그리기 전, 먼저 모델이 예측을 내리는 방식을 이해해야 합니다. 대부분의 분류 모델은 "1이다, 0이다"라는 단정적인 결과 대신, **"1(양성)일 확률이 85%이다(0.85)"**처럼 확률값을 반환합니다.

이 확률을 0 또는 1이라는 최종 결과로 변환하기 위해 기준선인 **임계값(Threshold)**을 설정해야 합니다.

* **기본 설정:** 보통 0.5를 기준으로 삼습니다. (0.7이 나오면 1로 예측, 0.3이 나오면 0으로 예측)
* **임계값 변경:** 기준을 0.3으로 낮추면? 모델이 0.4라는 확률을 뱉었을 때, 이전(0.5 기준)에는 '0'으로 분류했지만 이제는 기준을 넘었으므로 '1'로 분류하게 됩니다. 

이처럼 임계값을 이리저리 바꿔가며 모델의 성능 변화를 기록한 것이 바로 다음의 커브들입니다.

---

### 2. ROC 커브 (Receiver Operating Characteristic Curve)

임계값을 0부터 1까지 점진적으로 변화시키면서, 모델의 **진양성률(TPR, True Positive Rate)**과 **거짓 양성 비율(FPR, False Positive Rate)**이 어떻게 변하는지 점을 찍어 연결한 선입니다.



* **Y축:** TPR (실제 질병이 있는 사람 중 몇 명을 찾아냈는가?)
* **X축:** FPR (정상인 중 몇 명을 환자라고 오진했는가?)

---

### 3. AUROC (Area Under the ROC Curve)

ROC 커브 아래의 **면적(Area)**을 계산하여 모델의 성능을 단일 숫자로 평가하는 지표입니다. 보통 줄여서 **AUC**라고도 부릅니다.

| AUROC 점수 | 모델의 성능 | 설명 |
| :--- | :--- | :--- |
| **1.0** | **완벽한 모델** | 오진(FPR)은 하나도 없이 정답(TPR)만 다 맞추는 이상적인 상태입니다. |
| **0.5 ~ 1.0** | **일반적인 모델** | 현실 세계에서 우리가 구축하는 대부분의 모델이 이 구간에 속합니다. 숫자가 1에 가까울수록 성능이 좋습니다. |
| **0.5** | **쓸모없는 모델** | 동전 던지기처럼 무작위로 찍는(Random Guess) 상태입니다. 그래프상에서 대각선 직선으로 나타납니다. |

---

### 4. PR 커브 (Precision-Recall Curve)와 클래스 불균형

ROC 커브가 널리 쓰이지만, 이전 레슨에서 배운 **클래스 불균형(Class Imbalance)**이 심한 데이터(예: 정상인 99.4%, 심장병 환자 0.6%)에서는 모델의 성능을 과대평가하는 착시를 일으킬 수 있습니다. 이때 사용하는 것이 **PR 커브**입니다.



* **축 구성:** 임계값 변화에 따른 **정밀도(Precision)**와 **재현율(Recall)**의 변화를 그래프로 그린 것입니다.
* **사용하는 이유:** PR 커브를 계산하는 공식에는 **진음성(TN, True Negative)**이 아예 들어가지 않습니다.
* **효과:** 모델이 압도적으로 많은 '정상인'을 '정상'이라고 쉽게 맞춘 것(TN)에 대해 점수를 주지 않기 때문에, 불균형한 데이터에서도 모델이 진짜 찾고자 하는 소수의 타겟(예: 심장병 환자)을 얼마나 잘 찾아내는지 편향 없이 평가할 수 있습니다.

---

### 💡 실무 적용 가이드 (ROC vs PR)

* 데이터의 양성(1)과 음성(0) 비율이 비슷하다면 ➡️ **ROC 커브 (AUROC)** 사용
* 데이터에 음성(0)이 압도적으로 많고 우리가 찾고자 하는 양성(1)이 매우 드물다면 ➡️ **PR 커브** 사용
