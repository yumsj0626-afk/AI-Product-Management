
# 프로젝트 개요: 공유 자전거 시간대별 수요 예측 및 재배치 최적화

### 1. 배경
학교까지 도보로 15분 거리이지만, 대중교통이 마땅치 않아 전동 공유 자전거를 자주 이용합니다.
서비스 자체는 편리하지만, 정작 자전거가 가장 필요한 아침 등교 시간이나 출근 시간에는 거주지 주변에서 자전거를 찾기 어려운 경우가 빈번합니다.
정검이 완료된 자전거들을 새벽에 학교나 업무 지구 쪽으로 배치하는 시스템 때문이며, 이러한 경험은 대부분의 출근 등교를 목적으로 아침시간에 공유 자전거를 사용하고 자하는 고객들에게 서비스에 대한 '가용성 신뢰도'를 떨어뜨립니다.
* 결과적으로 사용자는 정기적으로 아침마다 공유자전거를 이용할 의지가 있음에도 불구하고 정기 결제권 서비스 이용을 확신하지 못해 결제를 망설이게 되는 주된 원인이 됩니다.

### 2. 데이터 선정
* 위와 같은 불편함이 공유 자전거 시스템 전반의 고질적인 시공간적 수요 불균형에서 기인한다고 판단하여, UCI 머신러닝 저장소의 **Bike Sharing Dataset**을 선정했습니다.
* 본 프로젝트에서는 시간대별 쏠림 현상을 미시적으로 파악하기 위해, 시간 단위로 집계된 `hour.csv` 데이터를 중점적으로 활용합니다.
* 날짜, 시간, 날씨(기온, 습도, 풍속 등) 정보를 기반으로 특정 시간대의 자전거 대여량(`cnt`)을 예측하는 회귀(Regression) 모델을 구축하고자 합니다.

### 3. 문제 정의 및 핵심 목표
단순한 예측을 넘어, 실제 서비스 운영에 적용할 수 있는 데이터 파이프라인 구축과 모델 비교를 목표로 합니다.

* **목표 1: 운영 인력 및 자전거 재배치 의사결정 지원**
  * "어느 시간대에 어느 지역의 수요가 폭증하거나 감소하는가?"를 예측하여, 선제적인 재배치(Rebalancing) 전략의 근거를 마련합니다.
* **목표 2: 특성별 결합 전처리(Preprocessing) 파이프라인 경험**
  * **파생 변수 생성**: 날짜 데이터로부터 시간(`hour`), 요일(`dayofweek`), 월(`month`) 등 유의미한 시간 피처를 추출합니다.
  * **범주형 인코딩**: 날씨(`weathersit`), 계절(`season`), 요일(`weekday`), 시간(`hr`) 등의 범주형 변수에 원-핫 인코딩(One-Hot Encoding)을 적용합니다.
  * **수치형 스케일링**: 기온(`temp`), 습도(`hum`), 풍속(`windspeed`) 등 스케일이 다른 수치형 변수를 정규화/표준화하여 모델의 학습 효율을 높입니다.
* **목표 3: 다양한 회귀 모델 비교 및 개선 구조 학습**
  * **Baseline 모델**: 선형 회귀(LinearRegression)를 통해 기준 성능을 설정합니다.
  * **규제 모델**: 다중공선성 방지 및 일반화 성능을 위해 Ridge, Lasso를 적용합니다.
  * **비선형 모델**: 복잡한 환경 변수 간의 상호작용을 포착하기 위해 의사결정나무(DecisionTree)와 랜덤 포레스트(RandomForest)를 활용합니다.
  * **앙상블(선택)**: 환경 구성이 가능할 경우 XGBRegressor를 추가하여 성능의 최대치(Upper bound)를 확인합니다.

### 4. 평가지표
비즈니스적 관점과 기술적 관점의 지표를 종합적으로 활용하여 모델을 평가합니다.

* **결과 지표 (Outcome)**: 
  * 예측을 통한 선제적 배치로 '가용성 신뢰도'를 확보하고, 궁극적으로 정기 결제 전환율을 높이는 것을 비즈니스 목표로 합니다.
* **출력 지표 (Output)**:
  * **RMSLE & WAPE**: 자전거가 부족해지는 상황(Under-prediction)에 더 큰 패널티를 부여하고, 수요가 몰리는 핫스팟의 예측력을 중점적으로 봅니다.
  * **MAE / RMSE**: 실제 대여량과 예측 대여량 간의 절대적/평균적 오차 크기를 측정합니다.
  * **R² (결정계수)**: 모델이 전체 대여량 분산을 얼마나 잘 설명하는지 평가합니다.

### 5. 최종 산출물 (Deliverables)
프로젝트 수행 결과로서 다음의 산출물을 도출할 예정입니다.

1. **모델 성능 비교표**: Baseline부터 비선형 트리 모델까지의 MAE, RMSE, R² 결과를 한눈에 비교하는 테이블.
2. **Actual vs Predicted 산점도 (Scatter Plot)**: 예측이 실제 수요를 얼마나 잘 따라가는지 직관적으로 보여주는 시각화.
3. **테스트 구간 라인 플롯 (Line Plot)**: 특정 기간(예: 1주일) 동안의 실제 대여량과 예측 대여량의 추세를 겹쳐 그린 시계열 그래프.
4. **중요 피처 및 계수 해석 (Feature Importance / Coefficients)**: 모델 판단에 가장 큰 영향을 미친 변수(예: 출퇴근 시간, 강수 여부 등)를 도출하여 비즈니스 인사이트 제공.
